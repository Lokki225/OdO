# TooXTips v1.1 Roadmap

**Version:** 1.1  
**Status:** Post-MVP Planning  
**Target Release:** Q3 2026 (estimated 8-12 weeks after MVP launch)

---

## Overview

TooXTips v1.1 builds on the MVP foundation (Agenda + Practice) by adding the Expenses module, voice commands, cloud sync, and advanced AI features. This roadmap prioritizes features by impact and implementation complexity.

---

## Strategic Pillars for v1.1

### 1. Complete the Trinity: Expenses Module

The MVP proved the core thesis with two modules. v1.1 completes it by adding the third pillar: spending patterns.

**Why now:**
- MVP users will have 2+ months of Agenda + Practice data
- The AI context model is proven; adding Expenses is an extension, not a redesign
- Cross-module insights become richer: "Your busiest work weeks correlate with highest spending. Automate non-essential expenses?"

**Scope:**
- Manual expense logging (form-based entry)
- Category breakdown (Food, Transport, Entertainment, Utilities, Other)
- Weekly/monthly spending summaries
- Budget tracking with alerts
- AI analysis of spending patterns

### 2. Voice Commands: Natural Language Input

MVP users will have experienced text-based AI interaction. v1.1 adds voice to reduce friction for quick logging.

**Why now:**
- Text input proves the concept; voice is an enhancement
- Abidjan context: voice commands reduce typing friction on mobile
- Accent/language variability can be tested with real MVP users

**Scope:**
- Voice-to-text for event creation ("Add meeting tomorrow 3pm")
- Voice-to-text for practice logging ("Log 30 minutes Japanese")
- Voice-to-text for expense logging ("Spent 5000 XOF food")
- Waveform feedback UI during recording
- Graceful fallback to text input if voice fails

### 3. Cloud Sync: Multi-Device Continuity

MVP is local-only. v1.1 adds optional cloud sync for users who want to access data across devices.

**Why now:**
- MVP users may want to check their schedule on desktop
- Cloud sync enables future features (sharing, collaboration)
- Firebase or iCloud provide turnkey solutions

**Scope:**
- Optional cloud sync (user can opt-in)
- Bidirectional sync (local changes sync to cloud, cloud changes sync to local)
- Conflict resolution (last-write-wins or user choice)
- Data encryption in transit and at rest
- Account management (sign-up, login, logout)

### 4. Advanced AI: Proactive Insights & Automation

MVP AI surfaces one suggestion per day. v1.1 AI becomes more proactive and actionable.

**Why now:**
- MVP users will have established patterns
- AI context is richer (Agenda + Practice + Expenses)
- Opportunities for predictive insights and automation

**Scope:**
- Weekly review summaries (AI-generated Sunday recap)
- Smart scheduling (AI proposes practice blocks in free slots)
- Spending insights ("You overspent on food by 20% this week")
- Habit anchoring (AI suggests recurring events for spontaneous patterns)
- Overcommitment alerts (with actionable suggestions, not just warnings)

## Feature Breakdown by Priority

### High Priority (Must Have for v1.1)

#### 1. Expenses Module (Full Implementation)

**User Stories:**
- As a user, I can log an expense with amount, category, and optional note
- As a user, I can view my spending by category (weekly/monthly breakdown)
- As a user, I can set a budget and receive alerts when approaching limit
- As a user, the AI can analyze my spending patterns and surface insights

**Technical Requirements:**
- SQLite schema for expenses (amount, category, date, note)
- Expense entry form (bottom sheet, similar to event/session entry)
- Expense summary view (category breakdown, spending trends)
- Budget tracking logic
- AI context integration (expenses included in context payload)

**Estimated Effort:** 5-7 days

---

#### 2. Voice Commands (MVP Quality)

**User Stories:**
- As a user, I can tap the microphone button and speak to create an event
- As a user, I can speak to log a practice session
- As a user, I can speak to log an expense
- As a user, if voice fails, I can fall back to text input

**Technical Requirements:**
- `speech_to_text` package integration
- Microphone permissions handling
- Waveform visualization during recording
- Voice-to-intent parsing (delegate to Claude)
- Graceful error handling (network, accent, background noise)

**Estimated Effort:** 4-5 days

---

#### 3. Weekly AI Review

**User Stories:**
- As a user, I receive a weekly summary (Sunday evening) of my week
- The summary includes agenda highlights, practice progress, and spending insights
- I can tap the summary to explore details or take action

**Technical Requirements:**
- Weekly notification trigger (Sunday 8pm)
- AI prompt for weekly review generation
- Summary UI (bottom sheet or modal)
- Action buttons (add event, log session, adjust budget)

**Estimated Effort:** 2-3 days

---

### Medium Priority (Should Have for v1.1)

#### 4. Cloud Sync (Firebase)

**User Stories:**
- As a user, I can optionally enable cloud sync in settings
- My data syncs automatically to the cloud
- I can access my data on multiple devices
- If I make changes offline, they sync when connectivity returns

**Technical Requirements:**
- Firebase project setup
- Authentication (email/password, Google sign-in)
- Firestore schema for events, sessions, expenses
- Bidirectional sync logic
- Conflict resolution strategy
- Data encryption

**Estimated Effort:** 6-8 days

---

#### 5. Smart Scheduling

**User Stories:**
- As a user, the AI proposes practice blocks in my free calendar slots
- I can tap a proposal to add it to my calendar
- The AI learns from my acceptance/rejection patterns

**Technical Requirements:**
- Enhanced free-slot detection (consider practice duration)
- AI prompt for scheduling suggestions
- Suggestion UI (inline in Practice slide or notification)
- Learning logic (track which suggestions user accepts)

**Estimated Effort:** 3-4 days

---

#### 6. Habit Anchoring to Recurring Events

**User Stories:**
- As a user, I can create recurring events in Agenda
- The AI suggests anchoring spontaneous practice sessions to recurring blocks
- I can confirm to formalize the pattern

**Technical Requirements:**
- Recurring event support (daily, weekly, custom)
- Recurrence rule storage (iCalendar format)
- Recurrence expansion (generate instances for calendar view)
- Pattern detection for habit anchoring

**Estimated Effort:** 4-5 days

---

### Lower Priority (Nice to Have for v1.1)

#### 7. Receipt Scanning

**User Stories:**
- As a user, I can take a photo of a receipt
- The app extracts the amount and category
- I confirm and the expense is logged

**Technical Requirements:**
- Camera integration
- OCR (Tesseract or cloud-based)
- Amount/category extraction
- Manual correction UI

**Estimated Effort:** 5-7 days (high complexity due to OCR variability)

---

#### 8. Home Screen Widget

**User Stories:**
- As a user, I can see my next 2-3 events on my home screen
- I can see today's practice status (streak, sessions logged)
- I can see a quick AI tip without opening the app

**Technical Requirements:**
- Flutter home screen widget support
- Widget configuration (size, refresh rate)
- Data sharing between app and widget
- Widget tap handling (deep linking to app)

**Estimated Effort:** 3-4 days

---

#### 9. Multi-Language Support (French)

**User Stories:**
- As a French-speaking user, I can use the app in French
- All UI text, dates, and currency are localized
- AI responses can be in French

**Technical Requirements:**
- Internationalization (intl package enhancements)
- French translation of all strings
- AI prompt engineering for French responses
- Testing with Ivorian French speakers

**Estimated Effort:** 4-5 days (translation + testing)

---

#### 10. Apple Watch / Wear OS Companion

**User Stories:**
- As a user, I can see my next event on my watch
- I can start a practice session from my watch
- I can receive AI notifications on my watch

**Technical Requirements:**
- WatchKit (iOS) or Wear OS (Android) integration
- Companion app UI (minimal, glanceable)
- Data sync between phone and watch
- Notification delivery to watch

**Estimated Effort:** 6-8 days

## Risk Mitigation for v1.1

| Risk | Mitigation |
|------|-----------|
| Voice commands fail on accents | Test extensively with Ivorian French speakers; provide fallback to text |
| Cloud sync introduces data loss | Implement robust conflict resolution; extensive testing before release |
| Expenses module abandoned | Design voice-first input; make logging < 3 seconds |
| Feature creep delays release | Strict scope control; defer lower-priority features to v1.2 |
| Performance degrades with 3 modules | Profile and optimize; defer non-critical features |
