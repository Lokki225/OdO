# Agenda Module — Data Implementation Plan

## Model: EventRow (Drift-generated from `Events` table — Story 1.4)

The `Events` table is already defined. The `EventRow` type is generated by Drift.

| Column | Dart Type | Drift Column | Notes |
|--------|-----------|-------------|-------|
| `id` | `int` | `integer().autoIncrement()` | Auto-generated; primary key |
| `title` | `String` | `text()` | Required |
| `startTime` | `int` | `integer()` | Epoch milliseconds UTC |
| `endTime` | `int` | `integer()` | Epoch milliseconds UTC |
| `category` | `String` | `text()` | `'personal'|'work'|'practice'` |
| `notes` | `String?` | `text().nullable()` | Optional |

> No schema changes needed — the `Events` table was created in Story 1.4.

## DAO: AgendaDao

**File:** `lib/features/agenda/data/agenda_dao.dart`

**Base class:** `DatabaseAccessor<AppDatabase>`

**Registration:** Add `AgendaDao` to `@DriftDatabase(daos: [..., AgendaDao])` in `app_database.dart`. Run `build_runner` after.

| Method | SQL Operation | Return Type | Story |
|--------|--------------|-------------|-------|
| `insertEvent(EventsCompanion companion)` | `INSERT INTO events` | `Future<int>` (generated id) | 2.1 |
| `updateEvent(EventsCompanion companion)` | `UPDATE events WHERE id = ?` | `Future<bool>` | 2.1 |
| `deleteEvent(int id)` | `DELETE FROM events WHERE id = ?` | `Future<int>` (rows affected) | 2.1 |
| `getEventById(int id)` | `SELECT * FROM events WHERE id = ?` | `Future<EventRow?>` | 2.1 |
| `getEventsBetween(int startMs, int endMs)` | `SELECT * FROM events WHERE start_time >= ? AND start_time < ?` | `Future<List<EventRow>>` | 2.1 |
| `watchEventsForDay(DateTime date)` | `SELECT * FROM events WHERE start_time >= dayStart AND start_time < dayEnd ORDER BY start_time` | `Stream<List<EventRow>>` | 2.1, 2.2, 2.3 |

**Day boundary computation:**
```dart
final dayStart = DateTime(date.year, date.month, date.day).toUtc().millisecondsSinceEpoch;
final dayEnd = DateTime(date.year, date.month, date.day + 1).toUtc().millisecondsSinceEpoch;
```

## Mapper: EventMapper

**File:** `lib/features/agenda/data/mappers/event_mapper.dart`

| Direction | Method | Logic |
|-----------|--------|-------|
| Row → Entity | `Event fromRow(EventRow row)` | Map each column; `DateTime.fromMillisecondsSinceEpoch(row.startTime, isUtc: true)`; validate category via `EventCategory.fromString(row.category)` |
| Entity → Companion | `EventsCompanion toCompanion(Event e)` | Map each field; `e.startTime.millisecondsSinceEpoch` for time columns; `Value(e.id)` for id (nullable) |

**Category parsing:**
```dart
static EventCategory _parseCategory(String s) => switch (s) {
  'personal' => EventCategory.personal,
  'work'     => EventCategory.work,
  'practice' => EventCategory.practice,
  _          => throw ArgumentError('Unknown EventCategory: $s'),
};
```

## Repository Implementation: AgendaRepositoryImpl

**File:** `lib/features/agenda/data/repositories/agenda_repository_impl.dart`

**Implements:** `AgendaRepository` (domain interface)

**Constructor injection:** `AgendaRepositoryImpl(this._dao)` where `_dao` is `AgendaDao`

| Method | Delegation | Error Wrapping | Story |
|--------|-----------|----------------|-------|
| `addEvent(Event event)` | `_dao.insertEvent(mapper.toCompanion(event))` | `try/catch` → `Failure(AppError.databaseWriteFailed)` | 2.1 |
| `updateEvent(Event event)` | `_dao.updateEvent(mapper.toCompanion(event))` | `try/catch` → `Failure(AppError.databaseWriteFailed)` | 2.1 |
| `deleteEvent(int id)` | `_dao.deleteEvent(id)` | `try/catch` → `Failure(AppError.databaseWriteFailed)` | 2.1 |
| `getEventById(int id)` | `_dao.getEventById(id)` then map row | `try/catch` → `Failure(AppError.databaseWriteFailed)` | 2.4 |
| `getEventsBetween(int startMs, int endMs)` | `_dao.getEventsBetween(startMs, endMs)` then map list | `try/catch` → `Failure(AppError.databaseWriteFailed)` | 2.1, 2.5 |
| `watchEventsForDay(DateTime date)` | Returns `_dao.watchEventsForDay(date).map((rows) => rows.map(mapper.fromRow).toList())` | No try/catch on stream (Drift emits errors as stream errors) | 2.1, 2.2, 2.3 |

**Note on `watchEventsForDay`:** Returns `Stream<List<Event>>` directly. The `AgendaRepository` interface declares this as `Stream<List<Event>>` (not wrapped in `Result`). Drift stream errors surface as `AsyncValue.error` in presentation.
